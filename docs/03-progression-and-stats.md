# 03 — Прогрессия и статы (формулы и баланс)

← [02 — Активности и квесты](./02-activities-and-quests.md) · далее → [04 — Экономика и магазин](./04-economy-and-shop.md)

Этот документ — «математика» игры: как активность превращается в числа, как считаются
уровни, и как держать баланс. Все константы вынесены в один раздел, чтобы их легко было
тюнить без переписывания логики.

---

## 1. Константы баланса (game config)

```jsonc
{
  "XP_BASE": 100,          // база кривой уровня
  "XP_EXP": 1.5,           // показатель степени кривой
  "STAT_BASE": 60,         // база кривой уровня стата
  "STAT_EXP": 1.4,
  "CRIT_CHANCE": 0.10,     // шанс крита по XP
  "CRIT_MULT": 2.0,
  "STREAK_TIERS": [        // нижняя граница дней → множитель
    { "days": 0,  "mult": 1.0  },
    { "days": 3,  "mult": 1.1  },
    { "days": 7,  "mult": 1.25 },
    { "days": 14, "mult": 1.4  },
    { "days": 30, "mult": 1.5  }
  ],
  "GOLD_STREAK_BONUS": 0.5 // макс. доп. множитель золота от стрика
}
```

> Эти значения хранятся в БД/конфиге, а не в коде, и могут меняться без релиза.
> Изменение коэффициентов не пересчитывает прошлый прогресс — только будущие начисления.

## 2. Кривая уровня персонажа

Сколько XP нужно, чтобы взять следующий уровень:

```
xp_to_next(level) = round(XP_BASE * level^XP_EXP)
```

Накопительный порог уровня = сумма по всем предыдущим:

```
xp_total(level) = Σ  xp_to_next(i),  i = 1..level-1
```

Пример (BASE=100, EXP=1.5):

| Уровень | XP за уровень | Всего XP к уровню |
|--------:|--------------:|------------------:|
| 1 → 2   | 100   | 100   |
| 2 → 3   | 283   | 383   |
| 5 → 6   | 1 118 | ~2 000 |
| 10 → 11 | 3 162 | ~13 600 |
| 25 → 26 | 12 500 | ~120 000 |
| 50 → 51 | 35 355 | ~700 000 |

→ ранние уровни «летят» (быстрый дофамин для новичка), дальше прогресс осознанно
замедляется (игра на годы). Тюнится через `XP_EXP`.

## 3. Кривая уровня стата

Аналогично, но мягче (`STAT_BASE=60`, `STAT_EXP=1.4`):

```
stat_to_next(stat_level) = round(STAT_BASE * stat_level^STAT_EXP)
```

Каждый стат копит собственные очки и берёт собственные уровни — независимо от общего.

## 4. Начисление за чек-ин

Псевдокод вычисления награды (источник истины для бэкенда):

```
function reward(activity, streakDays):
    base_xp   = activity.base_xp
    base_gold = activity.base_gold

    # 1) множитель за стрик
    streak_mult = lookup(STREAK_TIERS, streakDays)        # напр. 1.25

    # 2) крит (переменная награда)
    is_crit = random() < CRIT_CHANCE
    crit_mult = is_crit ? CRIT_MULT : 1.0

    # 3) опциональный множитель длительности (для активностей с duration)
    dur_mult = activity.has_duration
               ? clamp(duration_min / activity.ref_minutes, 0.5, 2.0)
               : 1.0

    xp   = round(base_xp * streak_mult * crit_mult * dur_mult)
    gold = round(base_gold * (1 + GOLD_STREAK_BONUS * (streak_mult - 1)))

    stat_points = round(base_xp * dur_mult)  # очки идут в стат активности

    drop = roll_item_drop(activity.rarity)   # см. 04-economy

    return { xp, gold, stat_points, is_crit, drop }
```

> Множитель экипировки (бусты XP от шмота) применяется поверх — см.
> [04 — Экономика](./04-economy-and-shop.md). Порядок: `base → стрик → крит →
> длительность → экипировка`.

## 5. Очки статов → уровень стата → класс

1. Активность добавляет `stat_points` в значение своего стата.
2. Значение стата конвертится в уровень стата по кривой из §3.
3. **Класс** пересчитывается как `argmax` по уровням статов (с порогом «равномерности»
   для *Авантюриста*) — см. [01 — Механики](./01-game-mechanics.md#4-классы--архетипы).

```
доминирующий = max(stat_levels)
если (max - min) <= 2:  класс = Авантюрист
иначе:                  класс = по доминирующему стату
```

## 6. Множитель стрика (детально)

`streakDays` — число подряд идущих дней с ≥1 чек-ином (с учётом активных заморозок).
Множитель берётся как наибольший tier, чья граница `days` ≤ `streakDays`.

```
streakDays = 9  →  tier {7: 1.25}  →  ×1.25
streakDays = 31 →  tier {30: 1.5}  →  ×1.5  (потолок)
```

Логика сброса/заморозки стрика — в [01 — Механики §5](./01-game-mechanics.md#5-стрики-серии-дней-).

## 7. Анти-абьюз и честность

- **Дневной потолок чек-инов на активность** (напр. 1 «спортзал» в день засчитывается
  с полной наградой, повторные — с убыванием), чтобы нельзя было «накликать» прогресс.
- **Кулдаун** на одинаковые чек-ины (минимальный интервал).
- Источник прогресса — только реальные чек-ины (нет внутриигровых «ферм»). Принцип честности
  из [видения](./00-vision.md#принципы-дизайна).

## 8. Что тюнить под ощущения

| Хочу… | Кручу |
|-------|-------|
| Быстрее старт | ↓ `XP_BASE` или ↓ `XP_EXP` |
| Дольше end-game | ↑ `XP_EXP` |
| Больше «вау» от чек-инов | ↑ `CRIT_CHANCE` / `CRIT_MULT` |
| Сильнее держать стрик | ↑ верхние tier-множители |
| Меньше инфляции золота | ↓ `base_gold`, ↑ цены в магазине ([04](./04-economy-and-shop.md)) |
