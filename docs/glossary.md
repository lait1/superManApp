# Глоссарий

← [12 — Дизайн персонажа](./12-character-design.md) · [README](../README.md)

Термины проекта **superMen**. Если встретил незнакомое слово в других документах — оно здесь.

---

| Термин | Значение |
|--------|----------|
| **Активность** | Атомарное реальное действие, которое отмечается чек-ином (спортзал, английский, работа…). Каталог в [02](./02-activities-and-quests.md). |
| **Ачивка (достижение)** | Постоянная награда-веха за пройденный рубеж. Не сбрасывается, копится в зале славы. |
| **Бары (прогресс-бары)** | Полоски заполнения (XP, статы, прогресс квеста). Всегда анимируются. |
| **Буст** | Эффект экипировки, увеличивающий XP/золото в домене. Ограничен потолком. См. [04](./04-economy-and-shop.md). |
| **Класс / Архетип** | Тип персонажа (Воин/Мудрец/Паладин/Друид/Бард/Авантюрист), вычисляется по доминирующему стату. См. [01](./01-game-mechanics.md). |
| **Крит** | Случайный ×2 к XP за чек-ин (шанс ~10%). Переменная награда. |
| **Core loop** | Петля вовлечения: действие → чек-ин → награда → возврат. См. [00](./00-vision.md). |
| **Daily report** | Ежедневная сводка достижений, приходит ботом в Telegram. См. [06](./06-notifications.md). |
| **Дроп** | Случайно выпавший предмет экипировки после чек-ина. |
| **Золото** | Внутриигровая валюта. Тратится только на виртуальную экипировку. См. [04](./04-economy-and-shop.md). |
| **initData** | Подписанные данные Telegram о пользователе; основа аутентификации без логина. См. [10](./10-telegram-mini-app.md). |
| **Инвентарь** | Купленные/выпавшие предметы пользователя; из него экипируют в слоты. |
| **Квест** | Цель поверх активностей с условием и наградой. Типы: daily/weekly/цепочка/side. См. [02](./02-activities-and-quests.md). |
| **Класс-квест** | Квест, привязанный к текущему классу персонажа. |
| **Level-up** | Повышение уровня персонажа; крупное событие с анимацией. |
| **Множитель стрика** | Коэффициент к XP, растущий с длиной серии дней. См. [03](./03-progression-and-stats.md). |
| **Ранг** | Крупная ступень престижа (Новобранец…Легенда), визуальный маркер поверх уровня. |
| **Reward event** | Ответ сервера на чек-ин со списком наград/событий; клиент проигрывает анимации. См. [09](./09-api.md). |
| **Слот** | Место экипировки (оружие/броня/амулет/аура/фон). |
| **Стат (характеристика)** | RPG-атрибут: 💪STR, 🧠INT, 🎯DIS, ❤️VIT, ✨CHA. Привязан к домену жизни. |
| **Стрик** | Серия подряд идущих дней с чек-ином. 🔥 |
| **Streak Freeze (заморозка)** | Расходник за золото, спасающий стрик при пропуске дня. |
| **Сочность (juice)** | Эмоциональная отдача интерфейса: анимации, конфетти, звук, вибрация. См. [05](./05-ui-ux.md). |
| **TMA (Telegram Mini App)** | Веб-приложение, работающее внутри Telegram. См. [10](./10-telegram-mini-app.md). |
| **Paper-doll** | Система отображения персонажа из накладываемых слоёв (база + предметы по слотам). См. [12](./12-character-design.md). |
| **Пиксель-арт (HD)** | Выбранный арт-стиль: пиксельные спрайты 128×192, фикс-палитра, PNG-слои. См. [12](./12-character-design.md). |
| **Palette-swap** | Пиксельный приём: один спрайт, разные палитры — так красим персонажа по классу. См. [12 §9](./12-character-design.md#9-канал-прогресса-3--цвет-по-классу-palette-swap). |
| **Рампа (ramp)** | Набор из 3–4 тонов одного цвета (тень→база→свет→блик) в пиксельной палитре. См. [12 §3](./12-character-design.md). |
| **Спрайт-лист (sprite sheet)** | Кадры анимации персонажа в одном PNG-атласе. См. [12 §10](./12-character-design.md). |
| **Эволюция силуэта** | Перерисовка базового спрайта героя на крупных вехах (рангах). См. [12](./12-character-design.md). |
| **Якорь (anchor)** | Фиксированная пиксельная точка на холсте 128×192 для стыковки слоёв. См. [12](./12-character-design.md). |
| **LoRA** | Дообученная модель для генерации ассетов в едином стиле/персонаже. См. [12 §11](./12-character-design.md#11-арт-пайплайн-ии-ассист-). |
| **Титул** | Текстовая награда-звание за квест/ачивку (напр. «Conquistador», «Полиглот»). |
| **Чек-ин** | Отметка о выполненной активности — главное действие пользователя. |
| **XP (опыт)** | Очки опыта; копятся в уровень персонажа и уровни статов. |
