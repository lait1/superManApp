# 11 — Роадмап

← [10 — Telegram Mini App](./10-telegram-mini-app.md) · далее → [12 — Дизайн персонажа](./12-character-design.md)

Поэтапный план от рабочего MVP к полноценной игре. Каждая фаза самодостаточна и
приносит ценность сама по себе.

---

## 🎯 Принцип

Сначала замкнуть **core loop** (действие → награда → возврат) на минимуме механик —
и только потом добавлять глубину. Без работающей петли остальное бессмысленно.

---

## Фаза 0 — Фундамент (инфраструктура)

> Цель: «скелет» проекта, по которому можно катать фичи.

- [ ] Go-бэкенд: каркас (`cmd/server`, `internal/{api,game,store,telegram,config}`), см. [07](./07-architecture.md).
- [ ] PostgreSQL + миграции, базовые таблицы и сиды каталогов, см. [08](./08-data-model.md).
- [ ] React (Vite) каркас, подключение Telegram WebApp SDK, тема, см. [10](./10-telegram-mini-app.md).
- [ ] Валидация `initData` + создание пользователя/персонажа.
- [ ] Деплой: HTTPS-домен, бот в BotFather, `docker compose`.

## Фаза 1 — MVP: замкнуть петлю

> Цель: можно отметить дело и получить дофамин. Это уже играбельно.

- [ ] Главный экран «Герой»: персонаж, 5 статов, бар XP, золото, стрик ([05](./05-ui-ux.md)).
- [ ] Ручной **чек-ин** активности (каталог из [02](./02-activities-and-quests.md)).
- [ ] Движок награды: XP, очки стата, золото, уровень, стрик ([03](./03-progression-and-stats.md)).
- [ ] Сочность: анимации +XP, level-up, конфетти, Haptic ([05 §6](./05-ui-ux.md#6-сочность-анимации-и-триггеры-награды-)).
- [ ] **Daily report** через Telegram-бота + cron-шедулер ([06](./06-notifications.md)).
- [ ] Настройки нотификаций + TZ.

**Критерий готовности MVP:** пользователь отмечает дела несколько дней подряд, видит рост
персонажа и получает вечернюю сводку в Telegram.

## Фаза 2 — v1: глубина игры

> Цель: причины возвращаться надолго.

- [ ] **Квесты**: daily / weekly / цепочки / side ([02](./02-activities-and-quests.md)).
- [ ] **Ачивки** + зал славы в профиле.
- [ ] **Магазин и инвентарь**: покупка/экипировка, эффекты шмота ([04](./04-economy-and-shop.md)).
- [ ] **Дроп шмота** при чек-ине (переменная награда).
- [ ] **Стрик-фриз**, тиры множителя, ранги.
- [ ] **Классы/архетипы** (вычисляемые) + титулы.
- [ ] Кастомные активности (пользователь добавляет свои).
- [ ] Крит-XP с анимацией.

## Фаза 3 — v2: ретеншн и социальность

> ⏸️ **ОТЛОЖЕНА.** Не начинаем, пока автор не протестирует и не подтвердит базовый
> функционал (Фазы 0–2) в реальном использовании: запуск в Telegram, петля
> чек-ин → награда, ежедневная сводка, квесты/магазин/прокачка. Только после этой
> валидации имеет смысл вкладываться в сезоны/соц-фичи.

> Цель: долгосрочное удержание и виральность.

- [ ] **Сезоны** (3 мес) с сезонным прогрессом и наградами.
- [ ] **Prestige** (опциональный сброс визуального уровня за престиж-метку).
- [ ] **Лидерборды / лиги**, друзья (через Telegram-контакты).
- [ ] **Гильдии / ко-оп квесты**.
- [ ] Интеграции автоматического чек-ина (Health/Strava/Duolingo и т.п.).
- [ ] Гибридные классы, скилл-деревья.
- [ ] Персонализированные мотивационные сообщения.

## Технический долг / постоянные задачи

- Наблюдаемость (логи, метрики чек-инов и доставки нотификаций).
- Тюнинг баланса по реальным данным (коэффициенты из [03 §1](./03-progression-and-stats.md#1-константы-баланса-game-config)).
- A/B тесты сочности и текстов нотификаций.
- Анти-абьюз чек-инов.

---

## Зависимости фаз

```
Фаза 0 (фундамент)
   │
   ▼
Фаза 1 (MVP: петля + daily report)   ◀── минимально жизнеспособно
   │
   ▼
Фаза 2 (квесты, ачивки, магазин, классы)
   │
   ▼
╔═══════════════════════════════════════════════╗
║  🚦 ГЕЙТ: автор тестирует базу в Telegram        ║
║     (петля, daily report, квесты/магазин)        ║
╚═══════════════════════════════════════════════╝
   │  только после подтверждения
   ▼
Фаза 3 (сезоны, соц-фичи, интеграции)   ⏸️ отложена
```
