# 06 — Нотификации (ежедневная сводка)

← [05 — UI/UX](./05-ui-ux.md) · далее → [07 — Архитектура](./07-architecture.md)

Нотификации возвращают пользователя в петлю. Доставляются через **Telegram-бота** —
это работает без отдельной авторизации и без браузерных push-разрешений, потому что
пользователь уже в Telegram.

---

## 1. Зачем

- **Возврат в петлю** (шаг ⑥ core loop из [видения](./00-vision.md#core-loop-петля-вовлечения)).
- **Актуализация результатов за день** — главный запрос: «что я сделал и куда двигаюсь».
- **Защита стрика** — напоминание, если за день ещё нет чек-ина.

## 2. Типы сообщений

| Тип | Когда | Цель |
|-----|-------|------|
| **Daily Report** | вечер (напр. 21:00 локального времени) | подвести итоги дня |
| **Streak Reminder** | если за день нет чек-ина и стрик под угрозой (напр. 20:00) | спасти стрик |
| **Morning Quest** *(опц., v1)* | утро (напр. 09:00) | подкинуть дневные квесты |
| **Milestone** | мгновенно при крупной вехе (LVL/ранг/легендарка) | усилить эмоцию |

## 3. Daily Report — контент

Сводка достижений за день. Пример сообщения бота:

```
🌙 Итоги дня, superMen!

Сегодня ты:
  🏋️ Спортзал        +63 XP  💪
  🧠 Английский       +50 XP  🧠  ⚡CRIT!
  💼 2 фокус-сессии   +100 XP 🎯

📈 Всего за день: +213 XP · 💰 +95 золота
🔥 Стрик: 12 дней (×1.25)
⚔️ Уровень: 14  ▓▓▓▓▓▓▓▓▓▓▓▓▓░░ 75%

🎯 Незакрытые квесты:
  ☐ Конкистадор — испанский (18/30 дней)

«Маленькие шаги каждый день побеждают большие рывки раз в месяц.»

[ Открыть superMen ]   ← кнопка → Mini App
```

Если за день **не было активности**:

```
🌙 Сегодня без чек-инов.

🔥 Твой стрик 12 дней под угрозой!
🧊 У тебя есть 1 заморозка — она спасёт серию,
   но лучше отметить хоть одно дело.

[ Отметить сейчас ]   ← кнопка → Mini App (чек-ин)
```

## 4. Streak Reminder

Отправляется только если: за «сегодня» нет чек-инов **и** стрик ≥ порога (напр. ≥ 3),
чтобы не спамить новичкам. Текст — короткий, мотивирующий, без чувства вины.

## 5. Расписание и реализация

```
┌─────────────────────────────────────────────────────────┐
│  Go backend                                               │
│                                                           │
│  ┌──────────────┐   тик раз в N минут   ┌──────────────┐ │
│  │  Cron-шедулер │ ────────────────────▶ │ Notification  │ │
│  │ (robfig/cron) │                        │   builder     │ │
│  └──────────────┘                        └──────┬───────┘ │
│                                                  │         │
│   выбрать пользователей, у кого                  ▼         │
│   локальное время == их слот                 ┌────────┐   │
│   и сегодня ещё не отправляли          ─────▶ │ Telegram│   │
│                                               │ Bot API │   │
│                                               └────┬────┘   │
└────────────────────────────────────────────────────┼───────┘
                                                       ▼
                                              сообщение в чат
```

Ключевые моменты:
- **Часовой пояс пользователя.** Telegram не отдаёт TZ напрямую → определяем при первом
  заходе из браузера (`Intl.DateTimeFormat().resolvedOptions().timeZone`) и сохраняем.
  Слот рассылки считается в локальном времени пользователя.
- **Идемпотентность.** Таблица `daily_reports` фиксирует, что за конкретную дату конкретный
  тип уже отправлен — повторно не шлём (см. [08 — Модель данных](./08-data-model.md)).
- **Шедулер тикает часто** (напр. каждые 5–10 минут) и на каждом тике выбирает тех, у кого
  локальное время попало в слот и сегодня сообщение ещё не уходило.
- **Отправка** — `sendMessage` Telegram Bot API с инлайн-кнопкой, открывающей Mini App
  (`web_app` button). Подробности бота — в [10](./10-telegram-mini-app.md).

## 6. Настройки пользователя

В профиле — управление нотификациями (без авторизации, привязано к Telegram-id):

```
┌──────────────────────────────────────┐
│  🔔 Нотификации                        │
│  Ежедневная сводка     [ вкл ]  21:00  │
│  Напоминание о стрике  [ вкл ]         │
│  Утренние квесты       [ выкл ]        │
│  Вехи (level/ранг)     [ вкл ]         │
└──────────────────────────────────────┘
```

## 7. Анти-спам правила

- Не более 1 daily report + 1 streak reminder в день.
- Streak reminder не шлём, если daily report уже ушёл и в нём была активность.
- Уважать выключенные тумблеры.
- Тихие часы: ничего не слать ночью пользователя.
