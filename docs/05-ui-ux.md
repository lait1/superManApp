# 05 — UI/UX: экраны, вайрфреймы, анимации

← [04 — Экономика и магазин](./04-economy-and-shop.md) · далее → [06 — Нотификации](./06-notifications.md)

Цель интерфейса: **минимум трения до чек-ина** и **максимум сочности** награды.
Приложение живёт внутри Telegram (Mini App), поэтому подстраивается под тему Telegram
и работает на телефоне в одну руку.

## 6. Сочность: анимации и триггеры награды 🎉

Самая важная часть UX — эмоция в момент награды. Каждое событие чек-ина возвращает
с сервера «reward event», который клиент проигрывает анимацией.

| Событие | Триггер / анимация |
|---------|--------------------|
| **+XP** | бар XP плавно дозаполняется, цифра `+50 XP` всплывает и тает |
| **Крит** | вспышка, цифра крупнее + золотом `CRIT ×2!`, отдельный звук |
| **Level-up** | пауза → вспышка → конфетти 🎉 → персонаж «пульсирует», цифра уровня перещёлкивается |
| **Ранг-ап** | полноэкранный оверлей, смена рамки/ауры, торжественный звук |
| **Дроп шмота** | анимация «сундук открывается» → карточка предмета влетает с цветом редкости |
| **Стрик +1** | пламя 🔥 разгорается, счётчик прибавляется с «щелчком» |
| **Заморозка сработала** | иконка 🧊 «спасла стрик», лёгкая успокаивающая анимация |
| **Квест выполнен** | чекбокс «штампуется» ✅, плашка награды съезжает сверху |
| **Прогресс-бары** | всегда анимируем заполнение, а не мгновенный скачок |

### Принципы анимации
- **Тайминг:** награда играется сразу после тапа, без задержек сети (оптимистичный UI,
  потом синк с сервером).
- **Не мешать:** анимации скиппаются тапом, не блокируют надолго.
- **Звук + лёгкая вибрация** (Telegram Haptic API) усиливают дофамин.
- **Редкое — ярче:** чем реже событие (крит, легендарка, ранг-ап), тем мощнее эффект.

### Технически (React)
- Анимации — **Framer Motion** (пружины, layout-анимации баров).
- Конфетти/частицы — лёгкая canvas-библиотека.
- Звук — предзагруженные короткие сэмплы; вибрация — `Telegram.WebApp.HapticFeedback`.
- Темизация — через CSS-переменные из `Telegram.WebApp.themeParams` (см. [10](./10-telegram-mini-app.md)).

## 7. Состояния и крайние случаи

- **Первый запуск:** короткий онбординг (3 экрана) → сразу главный экран (без логина).
- **Пустой день:** на главном — мягкий призыв «Отметь первое дело», без давления.
- **Обрыв стрика:** спокойное сообщение «Стрик сброшен, но прогресс с тобой. Начнём заново 💪».
- **Оффлайн:** чек-ин кладётся в очередь и синкается при появлении сети.
